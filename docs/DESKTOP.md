# Studio desktop shell

Studio Desktop is a thin Electron wrapper around the same `@studio/web` build used by the browser app. Image decoding, rendering, document history, project serialization, and exports remain in the shared client-side editor. The main process only owns operating-system integration.

## Local development

Run the web editor and Electron together from the repository root:

```sh
pnpm dev:desktop
```

Build an unpacked application for the current platform:

```sh
pnpm --filter @studio/desktop package:dir
```

Packaged builds load `apps/web/dist` from local resources. Vite therefore emits relative asset URLs; the hosted `/app` route and the `file://` route share the same bundle.

## Native boundary

The renderer has context isolation and Chromium sandboxing enabled. Node integration is disabled. A small preload bridge exposes only:

- native open and save dialogs;
- atomic file writes and recent-document paths;
- local scratch-cache controls;
- image clipboard writes and screen-colour sampling;
- operating-system menu commands and external-file-change notifications.

The editor receives file contents as `ArrayBuffer` values and never receives unrestricted filesystem or Node access. Normal browser drag and drop continues to use the web editor's import pipeline.

Project saves use a temporary file in the destination directory, flush it, and rename it into place. The shell watches the active project and reports external edits without reporting its own save as a conflict.

## Scratch storage

Scratch caching is optional. Documents estimated above 128 MB can serialize a recovery cache under Electron's per-user application-data directory. The default per-document limit is 2 GB and can be changed from **File > Desktop scratch storage…**. Clearing scratch storage never deletes source documents.

## Releases, signing, and updates

`.github/workflows/desktop.yml` packages macOS DMG/ZIP, Windows NSIS, and Linux AppImage/DEB artifacts. Tagging a commit with `v*` publishes electron-builder update metadata and artifacts to the `imlunahey/studio` GitHub release. `electron-updater` checks that channel only in packaged builds.

Signed macOS and Windows release jobs require these repository secrets:

- `MAC_CSC_LINK` and `MAC_CSC_KEY_PASSWORD`;
- `APPLE_ID`, `APPLE_APP_SPECIFIC_PASSWORD`, and `APPLE_TEAM_ID` for notarization;
- `WINDOWS_CSC_LINK` and `WINDOWS_CSC_KEY_PASSWORD`.
- `LINUX_GPG_PRIVATE_KEY` (base64-encoded) and `LINUX_GPG_PASSPHRASE` for signed Linux release checksums.

Linux packages are generated by the Linux runner with a signed `SHA256SUMS` manifest uploaded to the same release. Development workflow artifacts explicitly disable certificate discovery and are intentionally unsigned.
